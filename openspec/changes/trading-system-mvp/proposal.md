## Why

构建一个具备核心交易闭环的电商系统雏形，覆盖用户侧购物流程与管理后台运营能力，供演示和评审使用。当前尚无任何交易相关功能，需从零搭建。

## What Changes

- 新增用户端：手机号登录（mock 验证码 123456）
- 新增商品列表页：展示至少 5 件商品（从数据库读取）
- 新增商品详情页：展示图片、价格、描述
- 新增下单功能：mock 支付（点击按钮显示"支付成功"），订单数据写入数据库
- 新增订单列表页：用户可查看自己的历史订单
- 新增管理后台：固定账号 admin/admin123 登录
- 新增商品管理：查看列表、添加商品、编辑商品
- 新增订单管理：查看所有订单
- （加分）购物车功能
- （加分）商品搜索/筛选
- （加分）订单状态流转：待支付 → 已支付 → 已发货，管理后台可修改状态
- （加分）管理后台数据统计：销售额、订单数等

## Capabilities

### New Capabilities

- `user-auth`: 用户端手机号登录，mock 验证码固定为 123456；管理后台固定账号密码登录
- `product-catalog`: 商品列表与详情，支持从数据库读取，管理后台可增删改
- `order-management`: 用户下单（mock 支付）、订单持久化、用户侧订单列表、管理后台订单查看与状态管理
- `admin-portal`: 管理后台整体入口，含商品管理与订单管理页面
- `shopping-cart`: （加分）购物车能力，支持加购、查看、结算
- `dashboard-stats`: （加分）管理后台统计面板，展示销售额与订单数

### Modified Capabilities

（无现有能力变更）

## Impact

- 需新建前端工程（用户端 + 管理后台，可同仓库 monorepo 或分目录）
- 需新建后端服务（Go REST API，含 Swagger 文档）
- 需新建数据库 schema（MySQL，用户、商品、订单表）
- 引入 Redis 作为缓存层（token 黑名单、热点商品缓存等）
- 引入 Elasticsearch 作为商品搜索引擎
- 引入 RabbitMQ 作为消息队列（订单创建异步通知等）
- 无现有系统依赖，全量新建
